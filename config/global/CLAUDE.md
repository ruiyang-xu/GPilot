# Global Claude Code Instructions

## Who I Am

A finance professional using Claude Code as my primary operating system for investment research, deal analysis, portfolio management, and automation.

## Communication Preferences

- Default language: respond in the language I use. If I write Chinese, respond in Chinese. If English, respond in English.
- Be direct and concise. Skip pleasantries.
- When presenting analysis, lead with the conclusion, then supporting evidence.
- Use tables for structured data comparisons.
- For financial figures, use standard notation (e.g., $50M, 2.5x, 15% IRR).

## Output Defaults

- All generated documents should be professional, investor-grade quality.
- Research output is always bilingual (EN + CN). English first, then Chinese translation.
- When creating xlsx files, include clear headers and formatting.
- When creating docx/pdf, use clean professional formatting.

## Tools & Integrations

- Gmail: Draft-only mode. Never auto-send emails.
- Google Calendar: Read freely. Create events only when explicitly asked.
- Box: Use for document storage and retrieval.
- Perplexity: Preferred for deep research over basic web search.
- Vercel: For any web deployment needs.
- Feishu/Lark: Use lark-* skills for Feishu integration.

## Security

- Never expose API keys, portfolio details, or confidential deal data.
- All external-facing content requires my explicit approval.
- When in doubt, produce a draft and ask for review.
